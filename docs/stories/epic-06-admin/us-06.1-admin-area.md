# User Story: Área de Administração Central

## 🎯 Objetivo
Criar uma interface centralizada para administradores gerenciarem o workspace (tenant), usuários e configurações críticas do sistema, garantindo que usuários com roles inferiores não acessem dados sensíveis.

## 🧠 Contexto de Expert
Uma área de admin em um SaaS B2B não é apenas uma lista de usuários; é o "Cockpit" do negócio. Ela deve oferecer visibilidade sobre a saúde da operação (taxa de entrega de e-mails, status da conexão WhatsApp) e permitir o controle granular de quem pode fazer o quê (RBAC). A interface deve ser limpa, focada em dados e segura.

## 📝 Fluxo de Experiência/UX
1. **Acesso:** O usuário admin vê um link "Admin" no menu lateral.
2. **Dashboard Admin:** Visão geral de métricas do tenant (total de leads, usuários, mensagens enviadas no mês).
3. **Gestão de Usuários:** Lista de membros do time, convite de novos usuários e alteração de níveis de permissão (View, Op, Manager, Admin).
4. **Configurações Globais:** Ajustes de nome da empresa, plano atual e integração técnica.
5. **Logs de Sistema:** Visualização básica de erros ou eventos importantes para auditoria.

## 📝 Requisitos Técnicos
- **Proteção de Rota:** Uso de `src/lib/rbac.ts` para bloquear acesso de não-admins.
- **Backend:** Queries no Supabase filtradas por `tenant_id`.
- **Componentes:** Tabela de usuários, cards de métricas, formulários de configuração.
- **Mobile-first:** Layout responsivo para ajustes rápidos via celular.

## ✅ Critérios de Aceite
- [ ] Rota `/admin` criada e protegida por RBAC.
- [ ] Dashboard com métricas básicas do Workspace.
- [ ] Lista de usuários do Tenant com possibilidade de alterar roles.
- [ ] Seção de configurações do Workspace (nome, slug).
- [ ] Botão de acesso visível apenas para Admins no Sidebar.
