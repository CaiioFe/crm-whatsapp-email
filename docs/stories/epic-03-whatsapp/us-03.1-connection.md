# US 03.1 — Conexão de Instância e Teste de Mensagem

## 🎯 Objetivo (O que e por quê)
Permitir que o usuário conecte sua conta do WhatsApp ao CRM via Evolution API e valide a conexão enviando uma mensagem de teste para si mesmo ou para um lead.

## 🧠 Contexto de Expert
A conexão via Evolution API exige uma URL da API, uma Global API Key e um nome de instância. Precisamos gerenciar o estado do QR Code e garantir que a `api_key` seja armazenada de forma segura (neste MVP, usaremos o fluxo de configuração por tenant).

## 📝 Fluxo de Experiência/UX (Passo a passo do usuário)
1. O usuário acessa **Configurações > WhatsApp**.
2. Preenche os dados da Evolution API (URL e Key).
3. O sistema gera uma instância e exibe o QR Code para leitura.
4. Após conectar, o sistema exibe o status "Conectado".
5. O usuário acessa um Lead e clica em "Enviar WhatsApp".
6. Digita a mensagem e envia (validando o motor de disparo).

## 📝 Requisitos Técnicos
- Interface de configuração em `src/app/(dashboard)/settings/whatsapp`.
- Proxy de API para a Evolution API (para evitar CORS e expor keys).
- Tabela `whatsapp_provider_configs` populada corretamente.
- Integração básica no motor de jornadas para o step `send_whatsapp`.
